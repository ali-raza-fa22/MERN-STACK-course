## Part A: MongoDB (Database, Collections, Aggregations)

1. Design the bookstoreDB database with books, customers, and orders collections.
2. Create and insert at least 5 books into the books collection.
3. Develop a query to insert 3 customers into the customers collection.
4. Construct an order where a customer buys multiple books in one transaction.
5. Implement a query to retrieve all books priced above 30.
6. Design a query to find customers living in “Islamabad”.
7. Create an index on the title field of the books collection.
8. Develop a query to update the stock of a given book after an order is placed.
9. Construct a query to delete a customer with a specific email.
10. Implement an aggregation to count how many books each author has written.
11. Build a query to calculate the average price of all books.
12. Develop a query to show all orders placed in the last 7 days.
13. Construct a query to calculate total revenue grouped by customer.
14. Create a query that lists the top 3 best-selling books.
15. Design a query to join customers and orders collections using $lookup.
16. Develop a query to calculate the maximum price book in the collection.
17. Implement a query to list all books where stock is less than 10.
18. Build an aggregation pipeline to calculate total sales per month.
19. Create a query to group orders by city of the customer.
20. Design a query to remove all books with stock = 0.
